SMTP in Java

This is an SMTP email program, where a sender sends an email to a recipient. The program mimics how an SMTP protocol works by implementing a TCP protocol. The client sends commands to the Server which will act and reply to the client according to the command. Once the connection is terminated, the Server will write an email with the data received from client to a text file
Here is a list of the different commands that the Client can send to the Server in this program:

-CONNECTION: When the connection is first established, Client sends this command. Server replies with "220 "Server hostname""
-HELLO: Client sends its hostname to the Server. Server replies with "250 HELLO "Client hostname" pleased to meet you"
-MAIL FROM: The Client sends the senders email. The server stores the email and replies with "250 ok"
-RCPT TO: The Server sends the recipients email. The server stores the email and replies with "250 ok"
-DATA: The Client asks to send the text content of the email. The server replies with "354 End data with ." The Client then sends text and ends the text content by sending ".". Once Server receives "." it stops receiving "data" and replies with "250 ok message accepted for delivery".
-QUIT: Server replies with "221 "server hostname" closing connection" and closes the connection.
